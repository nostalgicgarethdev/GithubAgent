# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in $GitAgent — especially anything that could affect fee routing, dev address registration, or token launch integrity — please **do not open a public issue**.

Report privately by:
- Opening a GitHub Security Advisory in this repository (preferred)
- Or tagging an issue `security` with minimal details and we will follow up privately

We will respond within 72 hours and aim to resolve critical issues within 7 days.

---

## Scope

In scope:
- Fee routing exploits (wrong address receiving dev fees)
- Token launch manipulation (fake score inflation)
- Agent impersonation on X
- Smart contract vulnerabilities in fee distribution

Out of scope:
- X/Twitter platform security issues
- GitHub platform security issues
- Solana network-level issues

---

## Responsible Disclosure

We ask that you:
- Give us reasonable time to fix before public disclosure
- Not exploit the vulnerability beyond what is needed to confirm it
- Not access or modify other users' data

In return, we will:
- Acknowledge your report promptly
- Keep you updated on the fix
- Credit you publicly if you wish

# Contributing

Thanks for your interest in $GitAgent. Contributions are welcome.

---

## Ways to Contribute

- **Bug reports** — Open an issue with steps to reproduce
- **Documentation** — Fix errors, improve clarity, add examples
- **Feature suggestions** — Open a discussion with your idea
- **Code** — See below

---

## Opening Issues

Use the provided issue templates when available:
- Bug report
- Feature request
- Opt-out request (for developers whose repos were tokenized)

Please search existing issues before opening a new one.

---

## Pull Requests

1. Fork the repository
2. Create a branch: `git checkout -b feat/your-feature`
3. Make your changes
4. Commit with a clear message: `feat: add X` / `fix: Y` / `docs: update Z`
5. Push and open a PR against `main`

Keep PRs focused — one change per PR.

---

## Code Style

- Clear variable names over clever ones
- Comments for non-obvious logic
- No dead code in PRs

---

## Conduct

Be straightforward. No spam. No harassment.
